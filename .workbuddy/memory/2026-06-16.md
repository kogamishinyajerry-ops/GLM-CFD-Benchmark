## WSL/Ubuntu 修复（与 CFD-Benchmark 项目无关，开发环境事件）

### 故障
- Docker Desktop 报 `Wsl/Service/CreateInstance/MountVhd/HCS/ERROR_PATH_NOT_FOUND`
- docker cli 写 `~/.docker/config.json` 失败，exit 0xffffffff

### 诊断（确诊 Case A）
- `wsl -d Ubuntu -- echo ok` → 报 `MountVhd/ERROR_PATH_NOT_FOUND`
- 注册表 `HKCU\...\Lxss\{0053e06a-f113-4e3f-89a7-d13eabf10376}` 残留，DistributionName=Ubuntu, OsVersion=24.04, Modern=1
- BasePath 指向 `C:\Users\Kogami\AppData\Local\wsl\{0053e06a-...}\` 但**整个目录不存在**
- 全盘扫描 C/D 没找到任何 `ext4.vhdx` 备份 → 数据已彻底丢失

### 修复
- `wsl --unregister Ubuntu` → 残留注册表清理成功（虽然过程报 ERROR_UNHANDLED_EXCEPTION，列表里 Ubuntu 已消失）
- `wsl --install -d Ubuntu --no-launch` → 后台重装（task YbGz6X 仍在跑）

### 教训
- **WSL distro 迁移后不要清理"看似无用的"AppData\Local\wsl 目录**——这是 WSL2 自定义迁移路径，删了 = distro 数据全失
- `docker-desktop` 辅助 distro 不受 Ubuntu 故障影响，独立运行
- unregister 一个已损坏的 distro 报 `ERROR_UNHANDLED_EXCEPTION` 是正常的，列表状态会反映真实结果
